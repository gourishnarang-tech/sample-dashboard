How to read this dashboard:This dashboard shows all the users that have ever logged in. Users are tagged by DFG. Every user is classified into an engagement segment. Click any chart bar/slice/point to drill down. Click any user row to open their profile.

Segments explained:
Power User — 10+ active days AND still active in Jan/Feb 2026. Your champions who drive daily platform value.
Active User — 5–9 active days AND still active in Jan/Feb. Engaged but not yet habitual — prime candidates for power-user conversion.
Moderate User — 3–4 active days with some recent activity. Exploring but haven't committed to regular use.
Early Adopter — Churned — Joined Oct/Nov, used 3+ days, then stopped after Dec. Adopted early but something caused them to leave.
Low Engagement (1–2 logins) — Only signed in 1–2 times total. Onboarded but never returned.
Never Active (Account Live) — Have a live account but have never logged in. This is your biggest opportunity — 91 users.
Never Active (Disabled) — Account was created but is now disabled. No logins recorded.
Never Active (Not Live) — Account exists but was never set to live.

Metrics Matrix — derived from event logs. Each row is a user, each column is a metric. Heatmap intensity shows relative performance.

Active Days — How many days they logged in during this period.
Patient Details Viewed — How many times they opened a patient's care state screen (Level 1 depth).
Patient Subtasks List Viewed — Opened the eligibility screen for a patient+study combo. Deeper engagement than just viewing.
Patient Subtasks Criteria Viewed — Opened the detailed criteria screen (Level 2 depth). This is the deepest clinical review action.
Status Changes — Total times they changed a patient-study status (Recommended, Under Review, Rejected, Watchlist). This is the core action — it means they made a clinical decision.
Patient Screened per Active User — Status changes ÷ active days. Measures productivity — how many patients they process per day they're on the platform. Higher = more efficient.

New = unique new patient appointments. Follow-Up = unique returning patient visits.
Follow-Up >1 = unique patients with multiple follow-up visits.
Total Appts = New + Follow-Up + Follow-Up >1. Unique Patients = distinct patients (by MRN) seen that month.
Counts are based on unique appointment-patient keys (concate-key-appt-patient). Only rows where Consider appt = Yes are included. Use the Client Name dropdown to filter by client.
